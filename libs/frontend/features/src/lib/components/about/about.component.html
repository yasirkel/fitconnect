Casusbeschrijving

FitConnect is een webapplicatie waarmee sportclubs trainingen kunnen aanbieden en gebruikers zich kunnen inschrijven voor trainingen. Club-eigenaren kunnen hun clubs en trainingen beheren. Gebruikers kunnen trainingen bekijken en zich aan- of afmelden. De applicatie is gebouwd als full-stack SPA met Angular (frontend), NestJS (backend) en MongoDB binnen een Nx monorepo.

ERD / Datamodel

User (sterk)
- email (string)
- passwordHash (string)
- displayName (string)
- createdAt/updatedAt (date)

Club (sterk)
- name (string)
- address (string)
- city (string)
- description (string)
- sportsOffered (string[])
- ownerId (string)

Training (sterk)
- clubId (ObjectId)
- title (string)
- description (string)
- startTime (date)
- durationMinutes (number)
- capacity (number)

Enrollment (zwak)
- trainingId (ObjectId)
- userId (string)
- createdAt (date)

Relaties:
Club 1—N Training
Training 1—N Enrollment
User 1—N Enrollment
User ↔ Training is M:N via Enrollment

Functionele requirements

ID Functional Requirement Acceptatiecriteria Prioriteit (MoSCoW)
F-01 Als gebruiker wil ik een lijst van sportclubs kunnen bekijken, zodat ik weet welke clubs beschikbaar zijn.
  • Er wordt een lijst met clubs weergegeven.
  • Per club wordt minimaal naam en stad weergegeven.
 Must
F-02 Als gebruiker wil ik de details van een club kunnen bekijken, zodat ik info en trainingen kan zien.
  • Vanaf de clublijst kan ik naar een clubdetail navigeren.
  • In clubdetail zie ik clubinformatie en trainingen van de club.
 Must
F-03 Als gebruiker wil ik me kunnen inschrijven voor een training, zodat ik kan deelnemen.
  • In trainingdetail is een knop om in te schrijven.
  • Een gebruiker kan zich maar 1 keer inschrijven voor dezelfde training.
 Must
F-04 Als club-eigenaar wil ik clubs kunnen aanmaken/wijzigen/verwijderen, zodat ik mijn clubpagina kan beheren.
  • Mutaties vereisen login (JWT).
  • Alleen de owner kan update/delete (anders 403).
 Must
F-05 Als club-eigenaar wil ik trainingen kunnen beheren, zodat ik het trainingsaanbod kan aanpassen.
  • Mutaties vereisen login (JWT).
  • Alleen club-owner kan training create/update/delete (anders 403).
 Must
F-06 Als systeem wil ik verwijderacties blokkeren als er child-records bestaan, zodat data consistent blijft.
  • Club delete geeft 400 als er nog trainingen bestaan.
  • Training delete geeft 400 als er nog enrollments bestaan.
 Should
